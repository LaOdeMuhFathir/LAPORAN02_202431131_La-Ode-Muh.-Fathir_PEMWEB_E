Repositori ini berisi hasil pembuatan halaman web interaktif menggunakan HTML, CSS, JavaScript, serta penerapan Responsive Web Design (RWD). Proyek ini dibuat sebagai bagian dari Laporan Praktikum Pemrograman Web.

1. Struktur Dasar Halaman
   Buat satu file HTML dengan struktur sebagai berikut:
   a. Judul utama (<h1>) bertuliskan LAPORAN PRAKTIKUM
   b. Subjudul (<h2>) berisi Nama | NIM
   c. Tambahkan beberapa subbagian (<h3>) yang masing-masing berisi jawaban dari soal: 
      1.) Mengapa menggunakan CSS dan JavaScript dalam pemrograman web modern?
      2.) Jelaskan konsep dan tujuan utama dari Responsive Web Design (RWD)!
      3.) Bagaimana kombinasi CSS, JavaScript, dan prinsip RWD dapat menghasilkan tampilan web yang interaktif sekaligus adaptif? Jelaskan bagaimana ketiganya saling melengkapi!

3. Interaktivitas Menggunakan JavaScript
   a. Tambahkan fitur interaktif seperti: Saat pengguna mengklik subjudul (h3), tampilkan popup confirm() yang menanyakan apakah konten ingin disembunyikan. Jika “Ya”, sembunyikan teks di bawahnya; jika “Tidak”, biarkan
      tetap tampil.
   b. Buat tombol atau switch untuk mengganti tampilan menjadi dua mode:
      1.) Mode Rapi: desain nyaman dilihat, warna seimbang, teks mudah dibaca.
      2.) Mode Berantakan: warna mencolok/tidak serasi, tata letak acak, dan ukuran teks tidak proporsional.

4. Desain Tampilan Menggunakan CSS
   Gunakan CSS internal atau eksternal untuk mengatur tata letak dan gaya halaman:
   a. Gunakan Flexbox atau Grid untuk mengatur posisi teks dan elemen agar tampilan rapi.
   b. Atur warna latar belakang, jarak antar elemen (margin, padding), ukuran teks, serta efek transisi sederhana (transition) untuk memperhalus perubahan.
   c. Gunakan font yang mudah dibaca (misalnya sans-serif) dan hindari kombinasi warna yang menyilaukan.

5. Responsivitas dengan RWD (Responsive Web Design)
   Tambahkan media query (@media) untuk memastikan halaman tampil baik di berbagai ukuran layar:
   a. Layar desktop (≥1024px) → tampilkan dua kolom (misal: teks dan gambar berdampingan).
   b. Layar tablet (600–1023px) → ubah menjadi satu kolom dengan ukuran teks menengah.
   c. Layar smartphone (≤599px) → ubah menjadi satu kolom, ukuran teks dan tombol diperbesar agar mudah disentuh.
